# Fedora 安裝方式
---
```shell
# 查找
sudo dnf search riscv
# 然後下載需要的就好
# e.g. qemu-rsicv
```

### GDB
```shell
# 為了讓 gdb 支持 python 需要添加編譯選項
sudo dnf install gcc make python3-devel texinfo gmp-devel mpfr-devel
git clone https://sourceware.org/git/binutils-gdb.git && cd binutils-gdb
./configure --target=riscv64-unknown-elf --with-python --prefix=/home/liam/.local/riscv
make -j$(nproc)
make install

# dashboard
wget -P ~ https://github.com/cyrus-and/gdb-dashboard/raw/master/.gdbinit
pip install pygments
```


# 其他環境
---
### Qemu (不用)

```shell
sudo zypper install git-core gcc-c++ clang make qemu glib2-devel libpixman-1-0-devel ninja python-rpm-packaging

wget https://download.qemu.org/qemu-9.0.0.tar.xz
tar -xJvf qemu-9.0.0.tar.xz
cd qemu-9.0.0
mkdir build
../configure --target-list=riscv64-softmmu,riscv64-linux-user
ninja -j$(nproc)
```
* 若顯示 `no module name: packaging` 請下載 `python-rpm-package`
* 若顯示 `ERROR: glib-2.48 gthread-2.0 is required to compile QEMU` 請下載 `libglib-2_0-0`
* 若顯示 `ERROR: pixman >= 0.21.8 not present` 請下載 `libpixman-1-0-devel`

### Rust
###### nightly
1. 方式一: `rustup install nightly`
2. 方式二: 建立文件 rust-toolchain，裡面寫 `nightly` (不推薦!!!)

###### 相關軟體
```shell
# target platform
rustup target add riscv64gc-unknown-none-elf
cargo install cargo-binutils
rustup component add llvm-tools-preview
rustup component add rust-src
```

> binutils
> 是一組用來處理二進制文件的工具軟體集合，包含 as (匯編器), ld (鏈接器), nm (符號查看器), objcopy (複製器，可用來格式準換), gdb (調適器), objdump (反匯編) 與 readobj (可詳細讀取二進制文件), lldb (llvm 的 gdb), lld (llvm 的 ld)

### RISC-V toolchain (不用)
ref: https://github.com/riscv-collab/riscv-gnu-toolchain
```shell
sudo zypper in libexpact-devel
git clone https://github.com/riscv/riscv-gnu-toolchain
cd riscv-gnu-toolchain

./configure --prefix=/home/liam/.local/bin
make -j$(proc)
```
