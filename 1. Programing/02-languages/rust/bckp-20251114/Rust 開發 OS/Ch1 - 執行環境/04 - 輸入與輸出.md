# 使用 SBI 服務
---
我們編寫的 OS 為 RISC-V 中的 Supervisor 特權級，而 RustSBI 位於 Machine 特權級，兩者交互的介面稱為 Supervisor Binary Interface (SBI)。而 SBI 要提供的功能標準由 [SBI spec](https://github.com/riscv-non-isa/riscv-sbi-doc) 來規定。

### RustSBI
RustSBI 是使用 Rust 語言按照 SBI spec 標準實現的 Supervisor Execution Environment(SEE)，其他還有 OpenSBI (c 語言實現)。

內核因為沒有與 RustSBI 連接，而是分別的執行檔，故內核無法透過函數調用(符號地址)的方式來使用 SBI 服務，而 RustSBI 提供 `sbi_rt` 提供 SBI 服務接口，我們可以直接調用。
