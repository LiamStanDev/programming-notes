> 附註：異步 IO 
> 概念：
> - `libaio`（Linux asynchronous I/O library）是 Linux 提供的非同步 I/O 函式庫，允許應用程式在不阻塞的情況下執行 I/O 操作。
> - `io_uring` 是 Linux Kernel 5.1 版本引入的新非同步 I/O 介面，它相較於傳統的非同步 I/O（如 `libaio`），提供了更高效的 I/O 操作機制。使用了環形緩衝區（ring buffer），允許應用程式將 I/O 請求直接提交到核心空間，並從核心獲取完成的 I/O 請求。它減少了系統呼叫的次數。
> - `epoll` 是 Linux 核心提供的一種 I/O 多路複用機制，用於監視多個檔案描述符的 I/O 事件。
> 
> 實現機制：
> - `libaio` 使用非同步 I/O 系統呼叫來提交和獲取 I/O 請求。
> - `io_uring` 使用環形緩衝區和更高效的系統呼叫來提交和獲取 I/O 請求。
> - `epoll` 使用事件驅動模型，透過系統呼叫來管理和等待 I/O 事件。
> 
> 差異:
> `libaio`, `io_uring` 都是 linux 平台提供的非同步 io 功能，`epoll` 為監聽文件描述符進行事件通知。
