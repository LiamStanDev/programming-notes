# Java 運行機制
---
### JVM - Java Virtual Machine
為 Java 的運行時，負責運行 Java 程序，包含指令執行、內存管理與寄存器管理。

### JDK and JRE 
* JRE: Java Runtime Environment
	* 包含了運行Java程序的環境
	* $JRE=JVM+liberies$
* JDK: Java Development kit
	* $JDK = JRE + development\: tools$
	* 工具包含: javac, javadoc, javap等

##### 字符編碼
* ASCII: 使用128個字符，通常使用1 byte存儲
* Unicode: 為一個容納所有字的集合，容納全部需要4 bytes，有以下幾種實現。
	* UTF-8: 為可變長度編碼，使用**1 ~ 4 bytes**
		* ASCII 字符（U+0000 到 U+007F）只需要 1 字節。
		- U+0080 到 U+07FF 的字符需要 2 字節。
		- U+0800 到 U+FFFF 的字符需要 3 字節。
		- U+10000 到 U+10FFFF 的字符需要 4 字節。
	- UTF-16: 可變長度編碼，**2 ~ 4 bytes**
		- 大部分常用字符（U+0000 到 U+FFFF) 使用 2 字節。
		- 較少用的字符（U+010000 到 U+10FFFF）使用 4 字節。
	- UTF-32: **固定長度**編碼方式，每個字符都使用 4 字節

# 類型轉換
---
### 默認類型轉換
char → int → long → float → double
byte → short → int → long → float → double
##### 細節
1. char 不能轉換成 byte, short，反之亦然
2. char, byte, short各自或者相互進行運算時，會被轉換為int之後在運算
> char, byte, short只用來存儲，不用來計算。

### String與基本類型轉換
#### String to Basic
``` java
String s = "123"; 
int num1 = Integer.parseInt(s); 
String s2 = "0.2345"; 
double num2 = Double.parseDouble(s2); 
Boolean num3 = Boolean.parseBoolean("true"); 
char chr = s.charAt(0); // 取得第一個字符
```
#### Basic to String
* 使用 + 就行
```java
int num = 10;
String s = 10 + "";
double d = 22.55;
String s2 = d + "";
```
