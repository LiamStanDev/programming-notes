### Spring Boot
Spring框架是用來簡化企業在web開發的複雜度而創建的，採用以下策略:
1. 基於beam輕量級框架
2. 通過IOC與DI實現鬆耦合
3. 基於AOP和模板減少樣式代碼

Spring boot就是java web框架，可以簡化SpringMVC的使用，將約定俗成的配置簡化(**約定大於配置**)，能快速開發web應用。Spring Boot基於Spring，其作用旨在提升Spring框架的使用體驗，並集成大量常用的第三方庫配置(e.g. Redis, MongoDB, RabbitMQ...)。

### 微服務
隨者使用量增大，不能將整個程式只放在一台電腦中運行，故出現分部式架構，而Microservices就是秉承分部式概念而誕生的一種架構風格，將一個應用拆分成一系列微小的服務的組合，通過網路協議(http, rpc)進行溝通，每個服務都是可替換的與可獨立升級的程式。
#### 甚麼式單體服務(Monolithic Architecture, All in one)?
不將應用進行拆分如MVC架構等，將所有內容放入一個包中，可以方便單人開發、測試與部屬，若要放入多個服務器可以使用均衡負載(load blancer)就行。
缺點: 只要修改就需要停掉整個服務，並重新打包，且多人協同時會有很大的溝通問題。
