# Git commit
---
#### 種類
- feat：新增或修改功能（feature）
- fix：修補 bug（bug fix）
- docs：文件（documentation）
- style：格式
    - 不影響程式碼運行的變動，例如：white-space, formatting, missing semi colons
- refactor：重構
    - 不是新增功能，也非修補 bug 的程式碼變動
- perf：改善效能（improves performance）
- test：增加測試（when adding missing tests）
- chore：maintain
    - 不影響程式碼運行，建構程序或輔助工具的變動，例如修改 config、Grunt Task 任務管理工具
- revert：撤銷回覆先前的 commit
    - 例如：`revert：type(scope):subject`

#### commit message 如下
```text
[feat] add hotreload for config 
```

# Git flow
---
將 Branch 分成：
* master: 主要分支，爲穩定可使用的版本，只能透過 dev 或者 hotfix 進行 merge 來更新不能直接修改
* dev: 爲開發分支，也只能透過 merge 對該分支進行更新
* feat: 從 dev 分支拉出來，用於開發新的功能，名稱與開發的內容有關，開發完成並進行測試後之後可以合併進 dev 分支
* hotfix: 從 master 分支拉出來，用於 master 臨時出現問題需要進行即時修正，完成之後要同時合併回 master 與 dev 分支。
* release: 分
