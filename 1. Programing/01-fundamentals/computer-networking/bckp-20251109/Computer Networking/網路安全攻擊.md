## CSRF (Corss-Site Request Forgery)
---
### 概念
跨站請求僞造中，攻擊者誘導受害者訪問一個第三方網站，在受害者不知情下提交表單、更改用戶設置等。這種攻擊通常發生在受害者已經通過身份驗證的網站上。

### 流程
1. 用戶登入網站：受害者登入網站 A 在瀏覽器留下登入憑證 e.g. Cookies
2. 誘導訪問惡意網站：惡意網站包含一些請求指向網站 A
3. 自動發起請求：惡意網站自動發送請求向網站 A，由於瀏覽器會自動攜帶 Cookies，網站 A 會認爲這是受害者有效的請求。
4. 執行非本意操作：如更改密碼、轉賬等
### 防禦措施
1. Anti-CSRF Token: 網站爲每個用戶的會話生成一個唯一的隨機令牌，因爲攻擊者無法獲得這個令牌，所以無法構造有效請求。
2. 檢查 Referer Header：確認是否是信任的源，但可能被僞造
3. 使用 SameSite Cookie 屬性：限制 Cookies 只能在同源請求中發送。
4. 複雜的用戶交互
5. OAuth 2.0 Authorization Request 的 state 參數

## XSS (Cross-Site Scripting)
---
### 概念
跨站腳本攻擊，攻擊者將惡意腳本注入到內容中。分爲
1. 儲存型：將惡意腳本提交到目標網站的數據庫中，e.g. 評論、文章中，在瀏覽這些數據時，惡意腳本會被執行。
3. 反射型: 誘使用戶點擊一個連接，這個連接包含惡意腳本
4. DOM型：利用 DOM 來改變頁面內容，注入惡意腳本

### 防禦措施
1. 輸入過濾：特別注意會被嵌入到 HTML、Javascript 的內容進入數據庫。
2. 輸出編碼：避免瀏覽器誤解爲可執行代碼
3. Cookies HttpOnly
4. HTTP header `Content-Security-Policy`


## SQL Inject
---
### 概念
攻擊者在應用程式的輸入中插入惡意的SQL程式碼，當應用程式將這些輸入提交給資料庫時，這些惡意程式碼就會被執行，攻擊者可以利用這些執行的結果來進行未授權的資料存取或操作。

### 防禦措施
1. 使用參數化查詢
2. 對所有輸入進行嚴格的驗證
3. 限制資料庫的權限等。

## DDoS (Distributed Denial of Service)
---
### 概念
分佈式拒絕服務攻擊，攻擊者利用多個被控制的電腦對目標網站或服務發動大量請求，這種攻擊通常不是為了竊取資料或破壞內容，而是為了**讓目標服務的資源消耗殆盡**，導致合法使用者無法存取。

### 類型
1. Volumetric 攻擊：攻擊者發送大量數據包或請求來飽和網路帶寬。
2. 協議攻擊：
	1. SYN Flood: 攻擊者發送大量的TCP/SYN包到目標伺服器的端口，但故意不完成三次握手過程。伺服器為每個未完成的連接分配資源並等待完成握手，從而消耗伺服器資源。
	2. Ping of Death: 攻擊者發送大於正常大小的ICMP（網際網路控制訊息協定）資料包給目標。由於資料包大小超出了標準值，可能會導致目標系統崩潰或重新啟動。
3. 應用層攻擊：
	1. HTTP Flood: 攻擊者發送大量的HTTP請求到一個或多個目標伺服器上的網頁。這些請求本身是合法的，但數量龐大且頻率很高，導致伺服器無法處理合法使用者的請求。
	2. Slowloris: 攻擊者嘗試保持多個連接到目標伺服器上，並且每個連接都要求緩慢發送完整的請求頭。這導致伺服器資源被長時間佔用，阻止其他正常的連接請求。
	3. DNS Amplification: 攻擊者發送較小的查詢到DNS伺服器，但在查詢中格式來源位址為目標伺服器的位址。DNS伺服器回應這些查詢，並且較大的回應傳送到目標伺服器。
### 防禦措施
1. 流量過濾：使用訪火牆
2. 增加帶寬
3. 分佈式架構：使得某些資源受到攻擊服務還是能進行
