
# 🚀 Prompt 模板大全

## 1. 角色扮演型

讓 AI 扮演專家，回答更專業。

```
你是一位【角色，例如：資深後端工程師 / 財務顧問 / 健身教練】。
請根據【需求 / 問題】提供建議。
要求：用【風格，例如：專業 / 親切 / 簡單易懂】的方式回答。
```

**例子**
👉 你是一位資深 Linux 系統工程師，請教我如何在 Fedora 上設定 GPU passthrough。要求：分步驟說明，並附上常見錯誤與排查方式。

---

## 2. 教學 / 解釋型

把難的東西變簡單。

```
請用【對象，例如：10 歲小孩 / 高中生 / 初學者】也能理解的方式，
解釋【主題】。
輸出格式：定義 → 重點 → 例子 → 應用。
```

**例子**
👉 請用高中生能懂的方式，解釋「Time Weighted Return（TWR）」是什麼，為什麼有時候總報酬是正數但 TWR 是負的。

---

## 3. 比較分析型

幫助你決策。

```
請比較【選項 A】和【選項 B】，從以下面向分析：
- 成本
- 效益
- 風險
- 使用情境
最後給我建議，哪一個更適合【我的需求】。
```

**例子**
👉 請比較「使用 Docker」與「使用 Podman」在 Fedora 環境下的差異，從資源佔用、安全性、社群支援、未來趨勢分析，最後建議我該選哪一個。

---

## 4. 任務規劃型

拆解複雜任務。

```
我要在【時間】內完成【目標】。
請幫我做一個行動計畫，包含：
1. 任務分解
2. 每個階段的里程碑
3. 需要的工具 / 資源
4. 潛在風險與解法
```

**例子**
👉 我要在 3 個月內準備英文面試，請幫我做一個學習計畫，分成聽力、口說、寫作、模擬面試四大區塊，並附上每週行動重點。

---

## 5. 寫作生成型

控制文風與格式。

```
請幫我寫一篇【文章類型，例如：部落格 / 演講稿 / 社群貼文】，
主題是【主題】。
要求：
- 字數大約【字數】
- 風格：【專業 / 幽默 / 故事化】
- 輸出格式：【分段 / 標題 / 清單】
```

**例子**
👉 請幫我寫一篇 600 字的部落格文章，主題是「如何用 AI 提升程式員效率」，語氣要親切幽默，並用 3 個小標題分段。

---

## 6. 程式開發型

讓 AI 幫你 debug 或優化。

```
這是我的程式碼：
【貼程式碼】
請幫我：
1. 找出可能的錯誤
2. 提供優化建議
3. 加上必要的註解
4. 寫出 3 個測試案例
```

**例子**
👉 這是我的 Python 程式碼：

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

請幫我優化成高效版本，並加上單元測試。

---

## 7. 資料整理 / 摘要型

快速理解大量內容。

```
這是【一段文章 / 資料 / 報告】：
【貼文字】
請幫我：
1. 摘要重點（不超過 5 點）
2. 分類主要議題
3. 給我一個 1 句話結論
```

**例子**
👉 這是一篇關於生成式 AI 的新聞，請幫我整理重點成 3 條，再用一句話告訴我這篇文章的核心結論。

---

## 8. 表格生成型

把複雜東西轉成表格。

```
請把以下資訊整理成表格：
【資料內容】
表格欄位：【欄位 1 / 欄位 2 / 欄位 3】
```

**例子**
👉 請把「KDE、GNOME、XFCE」三種桌面環境的差異整理成表格，欄位包含：資源佔用、特色、適合族群。

---

## 9. 創意發想型

突破思考框架。

```
請給我【數字】個點子，主題是【主題】。
要求：每個點子不超過【字數】，並解釋它為什麼有用。
```

**例子**
👉 請給我 10 個「如何在英文面試中展現自信」的小技巧，每個不超過 30 字。

---

## 10. 分步驟解題型

讓 AI 條理清晰。

```
請一步一步幫我解釋【問題】。
輸出格式：
1. 問題拆解
2. 解題步驟
3. 最終答案
```

**例子**
👉 請一步一步幫我解釋「為什麼在 Linux 中需要 initramfs」，最後總結一句話答案。
