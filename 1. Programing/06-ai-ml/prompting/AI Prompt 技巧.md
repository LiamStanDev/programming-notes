下面我幫你整理「如何寫出好的 prompt（提示語／提問方式）」的重點，並附上多種示例。你可以把它當作一套「prompt 工具包」，逐步練習、調整，讓 AI 幫你做的事愈來愈精準、有用。

## 一、寫好 prompt 的關鍵原則

以下是從多個資料與實務經驗整理出來的 prompt 工程（Prompt Engineering）要點：

| 原則                               | 說明                                                               |
| -------------------------------- | ---------------------------------------------------------------- |
| **明確具體（Be specific）**            | prompt 要把你要的「事情、範圍、細節」說清楚，避免 AI 一頭霧水。 ([DigitalOcean][1])        |
| **給上下文 / 背景**                    | AI 若理解你的情境、目標、角色，回答會比較貼近你的需要。 ([MIT Sloan TLT][2])               |
| **格式、輸出樣式指示**                    | 明示你要什麼格式：清單、表格、摘要、章節等等。 ([Atlassian][3])                         |
| **角色扮演 / 指派身份**                  | 告訴 AI 它是什麼角色（專家、老師、律師、記者…）來回答。 ([Atlassian][3])                  |
| **舉例 / few-shot 提示**             | 在 prompt 裡給 AI 一兩個你理想的範例，讓它模仿。 ([DigitalOcean][1])               |
| **分步驟 / 引導思考（Chain-of-thought）** | 若任務複雜，可以讓 AI 先「思考過程」或「一步步來」，不要直接跳答案。 ([prompthub.us][4])         |
| **限制與例外說明**                      | 告訴 AI 有哪些限制（字數、時間區間、不可採用的資源等等）。 ([prompthub.us][4])              |
| **允許不確定 / 要求來源**                 | 若 AI 不確定時可以回答 “我不確定 / 我查不到” 或引用來源以減少誤信息。 ([Business Insider][5]) |
| **用分隔符 / 標籤區隔段落**                | 用符號（如 “---”、標題、XML 標籤等）分隔說明、範例、你的指令，讓 AI 辨認不同區塊。                 |

## 二、不同類型任務的 prompt 模板與示例

以下是幾個常見任務類型，以及我設計給 AI 的 prompt 範例。你可以把這些範例當作起點，再依你自己的情境微調。

---

### 類型 A： **寫作 / 內容創作**

**模板：**

> 你是 **[角色身份]**（例如：行銷專家、小說家、科技評論員）
> 幫我寫一篇關於 **[主題]** 的 **[類型]**（文章 / 部落格 / 社群貼文 / 廣告文案）
> 要求：**[受眾、風格、字數、重點要點、語氣]**
> 請分段 + 加上副標題 / 範例 / 重點整理

**範例 1：**

> 你是內容行銷專家，請幫我寫一篇「如何使用 ChatGPT 提升工作效率」的部落格文章，目標讀者是對 AI 有興趣但沒技術背景的上班族。
> 要求：大約 700 字；語氣要親切易懂；內文分三段，每段含 2–3 個小標題；最後附總結與行動建議。

**範例 2：**

> 你是廣告撰稿者，請幫我寫一則 200 字以內的 Facebook 廣告文案，用來推廣一款線上語言學習課程。
> 語氣希望是年輕、有朝氣、有點幽默；重點要強調「一日一句」、「隨時練習」、「專屬學習計畫」。

---

### 類型 B： **分析 / 比較 / 意見**

**模板：**

> 我有這樣的情境 / 資料 **[背景說明]**
> 請幫我 **分析 / 比較 / 評估 / 建議** **[目標 / 問題]**
> 輸出要包含：**[優缺點、風險、結論與推薦]**
> 請用 **表格 / 列點 / 分段** 呈現

**範例 1：**

> 我在考慮買兩款筆電：A 型號 vs B 型號。
> A 的規格是：Intel i7 / 16GB / 512GB SSD，價格 NT$35,000
> B 的規格是：AMD Ryzen7 / 16GB / 1TB SSD，價格 NT$32,000
> 請幫我比較兩款的優缺點、適合族群、哪一款比較划算，最後給出建議。

**範例 2：**

> 請分析「遠端工作 vs 在辦公室工作」的優缺點與風險，對於科技業公司而言，哪一種模式比較可行？請列出 5 項比較面向（例如：成本、溝通效率、員工滿意度、設備需求、管理難度），再給總結建議。

---

### 類型 C： **規劃 / 任務拆解 / 輔助工具**

**模板：**

> 幫我做一個 **[計畫 / 時程 / 提案 / 工具清單]**，目標是 **[目標]**
> 要包括：**步驟 / 阶段 / 里程碑 / 所需資源 / 注意事項**
> 時間範圍：**[多久內完成]**
> 如果可以，也幫我列出可能的風險或挑戰

**範例 1：**

> 幫我做一個為期 3 個月的「個人品牌經營計畫」，目標是從零開始打造社群影響力。
> 請列出每月重點任務、每週要做的事、應用的平台、內容策略、可能風險，還要有 KPI 指標。

**範例 2：**

> 我想開一門線上課程，主題是「如何用 AI 寫作」。請幫我做課程大綱 + 每堂課標題 + 每堂課重點 + 推廣策略。
> 課程總共 8 堂課，每堂約 45 分鐘；課程要適合初學者；最後要有行銷文案建議。

---

### 類型 D： **編碼 / 技術 / 程式協助**

**模板：**

> 我在寫 **[語言 / 框架 /工具]** 程式，目的是 **[功能 / 任務]**
> 我有這段程式碼 / 這些需求 **[附上程式碼或需求描述]**
> 請幫我：**檢查 / 改錯 / 優化 / 補上註解 / 寫測試**
> 要求：**[格式 / 可讀性 / 執行效率 / 邊界情況]**

**範例 1：**

> 我有以下 Python 程式碼：
>
> ```python
> def factorial(n):
>     if n == 0:
>         return 1
>     else:
>         return n * factorial(n-1)
> ```
>
> 請幫我：
>
> 1. 加上錯誤處理（輸入負數、非整數的情況）
> 2. 改寫為非遞迴方式（迴圈方式）
> 3. 寫至少 3 個測試樣例

**範例 2：**

> 我用 JavaScript + React 在做一個待辦清單 App，想加入「拖拉排序」功能。可以幫我給一段範例程式碼或推薦現有套件，以及如何整合到我的 React component 裡面嗎？

## 三、進階技巧與策略

除了上述原則與範例，以下是一些進階策略，可讓你的 prompt 更強大、更靈活：

1. **Prompt 分段設計**
   把你的 prompt 切成多個段落：**指令 / 背景 / 範例 / 要求 / 格式**，每個段落用明顯符號分隔（如 `---`、`###`、```）可以幫助 AI 分辨。

2. **遞迴式 prompt（Prompt Chaining）**
   有些任務可以拆成多個子任務：先讓 AI 做第一步、再根據結果進行下一步。這樣能控制過程、降低錯誤。

3. **讓 AI 思考 / 列步驟再回答**
   在 prompt 裡加上：「先列出思考過程／步驟，再給結論」或「請一步步地推理」。可以得到更有邏輯的答案。 ([prompthub.us][4])

4. **限制回答長度 / 層級**
   有時你只要摘要、有時要深入。你可以在 prompt 裡指定：「請在 5 條以內」、「每條不超過 30 字」、「分 3 層標題」等。

5. **要求來源 / 引用 / 驗證**
   讓 AI 在回答中標示它的推理、引用或說「下面是假設／資訊來源」。這樣能幫助你判斷答案可信度。

6. **用對話補充 / 迭代改進**
   寫一次 prompt 得到答案後，你可以回去補充細節、修正方向，再問一次。大多時候 prompt 工程就是「寫 → 看 → 調整 → 再寫」的過程。

7. **懶人提示（Lazy Prompt）**
   有時候可以只給關鍵資訊，讓 AI 自己判斷。這在某些情況下是有效的。比如你把錯誤訊息貼進去，AI 往往就能理解你要幫你 debug。這是高階使用技巧。 ([Business Insider][6])

## 四、給你五個馬上可以用的 prompt 範本

下面這五個 prompt，你可以直接拿來用，改一改主題或細節就能派上用場：

1. **「懶人 prompt + 分析」**

   > 我貼給你一段文字（或數據 / 錯誤訊息 / 網頁內容）：
   > 【這裡貼內容】
   > 請幫我分析一下：
   >
   > * 這段主要在講什麼？
   > * 有哪些重點／結論？
   > * 有什麼建議或補充？
   >   請先列出思考過程再給答案。

2. **「角色＋任務＋格式」**

   > 你現在是我的產品經理助理，幫我擬一個新手用戶上手流程（Onboarding Flow）文案和步驟。
   > 請以流程圖＋文字說明方式呈現，每步驟要有標題與說明。

3. **「比較分析」**

   > 請比較 A 選項 vs B 選項（把你的兩個選擇放進來），從成本、效益、風險、可行性、時間五個面向分析，最後給出你的建議。

4. **「計畫 / 時間表」**

   > 我有 6 個月要完成「寫一本電子書」這個目標。請幫我做一個月曆式時間表（每月、每週要做什麼）＋進度檢查點 + 可能遇到的挑戰與備案。

5. **「教學 / 解釋」**

   > 請把 **[某個複雜概念／技術主題]** 用「對高中生也能懂」的方式說明。
   > 要包含：定義、背景、重點、與生活的關聯、簡單範例。

## 五、使用與練習建議

* **從簡單開始**：先從簡單的 prompt 練手，逐漸加入背景、格式、範例等複雜度。
* **觀察 AI 回答風格**：看 AI 是怎麼理解你的 prompt，哪些地方會誤解，再回頭調整。
* **版本比對（A/B 測試）**：你可以寫兩個不同版本的 prompt，讓 AI 回答後比哪一個效果比較好。
* **累積 prompt 庫**：你可以把你調整過、效果不錯的 prompt 收藏起來，以後變通使用。
* **多問「why / how」**：常在 prompt 裡加入「為什麼？」「怎麼做？」這些引導句子，讓 AI 不只是給答案，也給過程。
